# CI Hygiene Round 2 — discovery

Three accumulated CI-hygiene fixes. Discovery findings + the fix for each.

---

## 1. Golden job "version skew" — it's floating `3.x`, not a `stable-3.44.1` pin

**Premise correction (discovery shift):** there is **no `stable-3.44.1` pin anywhere**.
Every `subosito/flutter-action` step in the repo uses `flutter-version: '3.x'` +
`channel: stable` — `flutter-ci.yml` (analyze/codegen/test/**golden**/…),
`golden-rebaseline.yml`, `make-verify.yml`, `openapi-ci.yml`.

**Real root cause:** `3.x` is a *floating* spec — it resolves to the latest stable
3.x **at the moment each job runs**. Goldens committed by a `golden-rebaseline` run
weeks ago (on an older 3.x) are then compared by today's `golden` job (on a newer
3.x). Flutter patch releases change font hinting / shaping / rasterization, so the
pixels drift → the informational `golden` job is **perpetually red from toolchain
drift, not real UI regressions**. Local is `3.44.0`; the current latest stable 3.x
is `3.44.1` (what `3.x` resolves to today).

**Fix (concern 1):** pin an **exact** version so the baseline and the comparison
always use the same toolchain. `flutter-ci.yml` gets a workflow-level
`env: FLUTTER_VERSION: "3.44.1"` referenced by every job; the other workflow files
pin the same literal with a "keep in sync" comment (a workflow-level env can't cross
files). Bumping Flutter later = change the version in these 4 files, then
re-baseline. Coverage is **kept** (the job stays informational + version-aligned),
so it's a meaningful green-on-match / red-on-real-regression signal — not the
drop-the-job alternative.

## 2. `make lint` generated-file false positives from a git worktree

**Symptom:** `golangci-lint run` (= `make lint`, chained by pre-push `make verify`)
flags `internal/api/gen/{core,fin}/server.gen.go` — 3 errcheck + 2 gocyclo + 1 gosec
— **only when run from a git worktree**, forcing `git push --no-verify`.

**Discovery — it's worktree-path-specific:**
- From the **main checkout** (`Mopro-Shop/`): `golangci-lint run ./internal/api/gen/...`
  → **0 issues**. The files carry the standard header
  `// Code generated by …/oapi-codegen/v2 … DO NOT EDIT.` and golangci-lint v2's
  default `exclusions.generated: lax` excludes them correctly.
- From a **worktree** (`mopro-ci-hygiene/`, `mopro-coupon/`, …): the SAME files are
  reported with a `../Mopro-Shop/internal/api/gen/…` prefix and are **not** excluded.
  `go list` resolves the package to the correct worktree dir, so this is golangci-lint
  computing the issue's relative path against an out-of-tree base — and its
  generated-file exclusion (which keys off the path) then fails to match the
  `../`-prefixed path. Net: the header-based auto-exclusion is **not worktree-robust**.

**Fix (concern 2):** stop relying solely on header auto-detection — add an explicit
`linters.exclusions.paths: ["internal/api/gen"]` (the oapi-codegen output dir) to
`.golangci.yml`. golangci-lint path regexes are **unanchored substring** matches, so
`internal/api/gen` matches **both** `internal/api/gen/core/server.gen.go` (main) and
`../Mopro-Shop/internal/api/gen/core/server.gen.go` (worktree). `exclusions.generated:
lax` is also set explicitly (it's the default, but making it visible documents intent).
Result: `make lint` / pre-push pass on a clean tree from a worktree → no more
`--no-verify`. `internal/api/gen/` is 100% generated (regenerated by `make api-gen`;
never hand-edited), so full exclusion is correct.

## 3. Pending golden rebaselines

**Stale baselines:** the cart goldens (CT-09 filled-cart two-column + the new CT-03
coupon line, PR #182) plus any baselines carrying the floating-`3.x` skew from §1.

**Fix (concern 3):** re-baseline on the **pinned** `3.44.1` via the
`golden-rebaseline` workflow (`workflow_dispatch`, runs `flutter test
--update-goldens` on `ubuntu-latest` and pushes the regenerated `*.png`/`*.png.meta`).
Goldens are platform- and version-specific (Linux CI pixels ≠ local macOS pixels), so
this **must** run on CI on the pinned version — it cannot be done locally. Sequence:
land the §1 pin → run `golden-rebaseline` on `chore/ci-hygiene-2` → the bot commits
3.44.1 baselines → the `golden` job goes green on this PR.
