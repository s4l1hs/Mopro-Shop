## Mobile grid + paginated provider (PLP-03)

- `category_products_screen.dart` → `CatalogShell` (`lib/widgets/catalog/
  catalog_shell.dart`) → `_LoadMoreSection` = the **"Daha Fazla Ürün Gör" button**.
- `filteredProductsProvider` (`filtered_products_provider.dart`) exposes
  **`hasMore` + `loadingMore`** (`ProductsState`), and **`loadMore()` sets
  `loadingMore=true` synchronously before the await** (:39) → the in-flight guard
  already prevents duplicate fetches. **No provider change needed** (Outcome A) —
  the scroll trigger just calls the same `onLoadMore` the button calls, gated by
  `hasMore && !loadingMore`.
- `CatalogShell` is shared by mobile + desktop (desktop wraps it in `_buildWide`).
  → add an **`infiniteScroll` flag** (`= context.isMobile`): when true, wrap the
  `CustomScrollView` in a `NotificationListener<ScrollNotification>` that fires
  `onLoadMore` within **150px** of the bottom and **hides the load-more button**
  (keeps the spinner + error/retry). Desktop (`infiniteScroll:false`) is unchanged.
