Mobile-first marketplace API for Mopro Shop. Buyer app + seller panel.

## Money format
All monetary values are **integer minor units** (kuruş for TRY; coin-minor for TRY_COIN).
Every monetary field is accompanied by a `currency` string field.
Never use decimal values. Display layer divides by 100.

## Timestamps
All timestamps are **RFC 3339 strings** (e.g. `\"2026-01-15T10:00:00Z\"`).
Dates (no time component) use ISO 8601 `YYYY-MM-DD`.

## Locale resolution
Product `title` and `description` are resolved server-side from the
`Accept-Language` request header. The response always contains a single
resolved string — never a `translations[]` array.
Fallback chain: requested locale → `tr-TR` → first available locale.

## Step-up authentication flow
Endpoints annotated with `stepUpAuth` require a short-lived step-up token
(TTL 5 min, claim `scope=high_sensitivity`). Sending a regular access token
to a `stepUpAuth` endpoint returns `403` with `code: step_up_required`.

Client flow:
1. Receive `403 step_up_required`.
2. Present OTP entry screen.
3. `POST /auth/step-up` with access token + fresh OTP code → `step_up_token`.
4. Retry the original request with `Authorization: Bearer <step_up_token>`.

## X-Trace-Id
Clients should generate a UUID per request and attach it as `X-Trace-Id`.
The server echoes the value in `error.trace_id`. Falls back to a
server-generated UUID if the header is absent.

## Idempotency
All state-mutating endpoints (POST/PUT/PATCH/DELETE, except OTP request)
require `X-Idempotency-Key` (UUIDv7). Duplicate keys within 24 hours return
the cached response without re-executing the operation.

- `/wallet/_*` → fin-svc
- `/cashback/_*` → fin-svc
- All other `/_*` → core-svc
